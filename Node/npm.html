包（package）

  • 包中必须包含一个package.json文件，用于发布到npm registry
  • 包可以是file或directory，可以归属于user或organization，也可以是private或public
  • 包可以是以下格式（formats):
    • 一个包含package.json和对应的程序的目录
    • 一个上述目录对应的gzip压缩文件
    • 一个registry上的标识
      • url地址
      • <name>@<version>
      • <name>@<tag>
      • <name>(默认latest)
    • 一个git仓库地址（默认master分支）

作用域（scope）

  • npm v2+版本支持作用域，以前缀@且后缀/的形式放在package名前，用于避免名称冲突
    • @npm/package-name (npm作用域下的包)
  • private包一定有scope，unscoped的包一定是public的
  • npm access public <package-name>将private包更改为public
  • npm access restricted <package-name>将public包更改为private

package.json

Specifics of npm’s package.json handling
Creating a package.json file

  • 创建
    • 在包目录下执行npm init
      • --yes选项按默认值生成
      • --scope=@scope-name生成scoped模块
    • ~/.npm-init.js文件可定制上述命令执行后生成的文件内容
    • 常见配置：
  • 字段：
    • name 由小写字母、-、_字符组成的包名，习惯用包所在的目录名
    • description 描述信息
    • version x.x.x格式的版本，默认1.0.0 Semantic Versioning 2.0.0
    • scripts 可执行脚本
    • dependencies 产品中的依赖，执行npm install时会自动下载，--save-prod选项自动添加package.json
    • devDependencies 本地开发环境中的依赖，执行npm install时会自动下载，--save-dev选项自动添加到package.json
    • keywords 检索时的关键字
    • author 推荐name <email> (website)格式
    • license 许可证
    • bugs bug信息
    • homepage 主页信息

发布（publish）

  • npm publish发布private或unscoped的包
  • npm publish --access public发布scoped的公共包
  • npm publish --tag <tag>发布到指定tag(默认latest)
  • npm dist-tag add <package-name>@<version> [<tag>]为指定的版本打tag

模块（module）

  • 模块是在node_modules目录下的文件或目录，可通过require()函数加载
    • node_modules/a.js （a文件是模块，不是包）
    • node_modules/b/ （b目录是模块，不一定是包）
    • node_modules/c/package.json （c目录是模块，也是包）
    • node_modules/@myscope/d/package.json （d目录是scope包）
      • package.json文件必须有main字段

javascript包管理器（package manager）
javascript公共数据库 https://registry.npmjs.org (default)

  • 包依赖
    • 当包依赖中有git地址，npm会自动调用git命令进行安装
    • 当包依赖中有本地待编译的C++模块时，通过node-gyp包可以自动编译它
    • 查看当前项目中的包依赖：npm explain <package>
  • 安装包: npm install
    • 本地安装：可通过require()调用它
      • 包被安装到当前项目的./node_modules
      • 可执行程序被安装到当前项目的./node_modules/.bin
      • 工作目录：遍历目录树，寻找包含package.json或node_modules的目录作为工作目录
    • 全局安装：可通过命令行调用它
      • 包被安装到 $npm_config_prefix/lib/node_modules
      • 可执行程序被安装到 $npm_config_prefix/bin
      • man文档被安装到 $npm_config_prefix/share/man
  • 链接包：npm link
    • 创建全局链接：cd <package>; npm link
      • 包的全局链接：{prefix}/lib/node_modules/<package> 指向 <project>
      • 可执行文件链接：{prefix}/bin/{name} -> <project>/bin/{name}
    • 链接全局包：npm link <package> 在当前项目下创建全局包的链接
  • 包缓存：npm cache
    • 缓存目录：~/.npm
  • 配置npm：通过npm config ls -l查看
    • 通过命令行选项：--<key> <val>（默认true）
    • 通过环境变量：export npm_config_<key>=<val>
    • 通过项目配置：<project>/.npmrc
    • 通过用户配置：~/.npmrc
    • 通过全局配置：$npm_config_prefix/etc/npmrc
