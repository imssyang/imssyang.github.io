RTMP协议
[ Media ]

Posted on 11/18/2021 Edited on 11/23/2022 Symbols count in article: 3.8k Reading time ≈ 3 mins.

RTMP (Real-Time Messaging Protocol) 最初被 Macromedia(2005 年被 Adobe 收购) 开发，作为一种私有协议用于在 Flash 播放器和服务器间双向传输音视频流，Adobe 在2009年部分开源了此协议。

Adobe’s Real Time Messaging Protocol (2012)

基本特性

  • 支持编码：
    • Audio: AAC, AAC-LC, HE-AAC+ v1&v2, MP3, Opus, Speex
    • Video: H.264, VP8, VP6, ScreenVideo_v1&v2
  • 优势：
    • 持久化连接（persistent connections）：默认 TCP/1935 端口
    • 低延迟通信（low-latency communication）：最小缓冲（minimal buffering），通常5秒左右延迟
    • 支持平滑传输（smooth streaming）：将流（stream）切分为片段（fragment，默认audio/64B，video/128B，可动态协商）
    • 支持多路复用（multiplexed）：多条流（stream）可以在同一个连接上交错传输。通常为包层级交错（packet-interleaved）而非片段层级（fragment-interleaved）。
    • 支持推流与拉流
  • 劣势：
    • 浏览器不再支持flash控件后，rtmp流必须经过转码后才能在浏览器播放。

交互流程

  • 6次握手
  • 建立连接
  • 推流

  • 基于一些虚拟通道（virtual channels）传输包数据：
    • channel: RPC 请求和应答 (基于 AMF - Action Message Format 定义消息体)；
    • channel: video 流；
    • channel: audio 流；
    • channel: 控制消息（例如片段大小协商）。

协议依赖

AMF协议

Augmented BNF for Syntax Specifications: ABNF (1997)
Action Message Format – AMF0 (2007)
Action Message Format – AMF0 (2013)
Online Binary Convert (IEEE754 Double precision 64-bit)

  • AMF0
  • AMF3
  • 一种紧凑的二进制格式，字符串采用UTF8编码；
  • 一个字节标识数据类型：
    • 0x00 Number (64bit双精度浮点数，网络字节序)
    • 0x01 Boolean (8bit无符号数，0x00:false; !0x00:true)
    • 0x02 String (16bit字符串长度，后跟UTF-8编码的字符串)
    • 0x03 Object (键值对的集合)
    • 0x05 Null
    • 0x08 ECMA Array (32-bit entry count)
    • 0x09 Object End (16bit空字符串长度作为前缀)
    • 0x0a Strict Array (32-bit entry count)
    • 0x0b Date (64bit双精度浮点数，带有16bit整数时区偏移)
    • 0x0c Long String (32bit字符串长度，后跟UTF-8编码的字符串)
    • 0x0f XML Document (32bit字符串长度，后跟UTF-8编码的字符串)
    • 0x10 Typed Object (16bit的UTF-8编码的名字长度，后跟条目)
    • 0x11 Switch to AMF3
  • 提高了二进制压缩率，支持更复杂的数据类型；
  • 一个字节标识数据类型：
    • 0x00 Undefined
    • 0x01 Null
    • 0x02 Boolean False
    • 0x03 Boolean True
    • 0x04 Integer (8+bit可扩展长度的整数)
    • 0x05 Double (64bit双精度浮点数)
    • 0x06 String (8+bit可扩展长度的整数，后跟UTF-8编码的字符串)
    • 0x07 XMLDocument (8+bit可扩展长度的整数，后跟UTF-8编码的字符串)
    • 0x08 Date (8+bit可扩展长度的整数，带有64bit双精度浮点UTC偏移时间)
    • 0x09 Array (8+bit可扩展长度的条目数)
    • 0x0A Object (8+bit可扩展长度的条目数)
    • 0x0B XML (8+bit可扩展长度的整数标志)
    • 0x0C ByteArray (8+bit可扩展长度的整数标志，具有可选的8位字节长度)
    • 0x0D VectorInt
    • 0x0E VectorUInt
    • 0x0F VectorDouble
    • 0x10 VectorObject
    • 0x11 Dictionary
