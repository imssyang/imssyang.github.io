CUDA基础
[ CUDA ]

Posted on 2/20/2022 Symbols count in article: 14k Reading time ≈ 13 mins.

cuda编程基础，其内容主要源自nvidia的编程指南，提取重要概念与设计，结合实例与测试数据于以说明。

CPU vs GPU

  • cpu被设计为在线程（thread）中执行一系列顺序（sequential）操作，可支持几十个线程并行执行
    • 更多的晶体管用于缓存大数据和复杂的流控制，以支持一系列顺序操作，内存访问延迟较高
  • gpu被设计为支持成千上万个并行执行序列，常用于分摊较慢的线程，以获取更大的吞吐量
    • 更多的晶体管用于数据处理（例如浮点计算），以支持高度并行（parallel）计算，内存访问延迟较低

CUDA模型

CUDA: A General-Purpose Parallel Computing Platform and Programming Model

  • 2006年被NVIDIA推出的并行计算平台和编程模型，可利用NVIDIA GPUs更好的解决许多复杂的计算问题
  • CUDA是一种可伸缩（scalable）的编程模型，随着处理器核心数的增加可透明的扩展并行性
  • CUDA为C程序员保持较低的学习曲线，在语言上扩展的核心概念有：
    • 线程组的层次结构（a hierarchy of thread groups）
    • 共享内存（shared memories）
    • 栅栏同步（barrier synchronization）

硬件设计

  • NVIDIA GPU架构是围绕一个可扩展的多线程流多处理器阵列（SMs）构建的
    • 一个多处理器（multiprocessors）被设计成可并发执行数百个线程（thread）
    • 线程块（thread block）或线程块中的线程（threads）均可在一个多处理器中并发执行
    • 采用SIMT (Single-Instruction, Multiple-Thread) 架构，小端（little-endian）表示

      1. 分离host code和device code
      2. 将device code编译成assembly form (PTX code) 和 binary form (cubin object)

语言扩展

  • FunctionExecutionSpace:
    • __global__：声明可被kernel调用的函数，只能返回void，且不能是类成员函数，只能从host调用它
    • __device__：声明在device上执行的函数，只能从device调用它
    • __host__：声明在host上执行的函数，只能从host调用它（可与__device__一起使用）
    • __CUDA_ARCH__：定义代码兼容的计算能力（SM）
    • __noinline__：提示编译器尽可能不要内联该函数
    • __forceinline__：强制编译器内联该函数
  • VariableMemorySpace:
    • __device__：声明驻留在device上的变量
    • __constant__：声明驻留在常量内存空间的变量
    • __shared__：声明驻留在一个线程块的共享内存空间的变量
    • __managed__：声明可被device和host代码直接读写的变量
    • __restrict__：告知编译器指针参数不存在别名，可以优化函数代码

计算模型

cuda-c-programming-guide

  • Kernels: __global__修饰的c++函数
    • 内建变量：
      • gridDim.x/y/z 一个grid中block的数量（各方向）
      • blockDim.x/y/z 一个block中thread的数量（各方向）
      • blockIdx.x/y/z 一个grid中的block索引（各方向从0开始）
      • threadIdx.x/y/z 一个block中的thread索引（各方向从0开始）
      • gridDim.x * blockDim.x 一个grid中线程的数量（x方向）
    • 调用语法：<<<blocksPreGrid, threadPreBlock>>>
  • ThreadHierarchy: grid -> block -> thread，各层级均有三个维度 (x, y, z)
    • 三个维度分别用于vector, matrix, volume的计算
      • 一维I(x)：ThreadID = I.x
      • 二维I(x,y)：ThreadID = I.x + I.y * D.x
      • 三维I(x,y,z)：ThreadID = I.x + I.y * D.x + I.z * D.x * D.y
    • 每个block是独立执行的：可以并行（in parallel），也可以串行（in series）
    • 一个block上的所有thread限制在同一块显卡上分配核心且共享显存，因此thread数量（<=1024）是有限的
    • 一个block上的所有thread之间通过SharedMemory共享数据，并通过Cooperative Groups API协调内存访问
      • __syncthreads()等待一个block内的所有thread都执行到此位置后，才允许继续执行（barrier同步）
  • MemoryHierarchy:
    • GlobalMemorySpace
      • global memory - access by grid0, grid1, …
      • per-block shared memory
      • per-thread local memory
    • ConstantMemorySpace
    • TextureMemorySpace
  • HeterogeneousProgramming:
    • cuda线程在协处理器（coprocessor）上运行，即c++程序中kernals并行跑在gpu上时，其余串行部分同时跑在cpu上
    • cpu和gpu分别跑在各自独立的内存空间中，并通过Unified Memory桥接它们
