unittest单元测试框架
[ Python ]

Posted on 1/1/2022 Edited on 1/16/2022 Symbols count in article: 8.7k Reading time ≈ 8 mins.

unittest(Unit testing framework)受到JUnit启发而设计，标准库原生支持。

基本命令

测试用例（test case）

  • 执行unittest.main()，运行测试用例
  • 继承unittest.TestCase，创建测试用例
  • 测试方法名必须前缀”test”
  • 使用TestCase.assert*()进行测试
  • app/widget.py
  • tests/widget/test_widget.py

测试治具（test fixture）

  • 每个测试方法之前执行TestCase.setUp()，之后执行TestCase.tearDown()。
    TestCase.setUp()执行测试对象的__init__()，在TestCase.tearDown()清理测试对象占用资源。
  • 在运行单个类的测试之前执行TestCase.setUpClass()，之后执行TestCase.tearDown()，必须用@classmethod修饰。
  • 在运行单个模块的测试之前执行TestCase.setUpModule()，之后执行TestCase.tearDown()，应实现为function。
  • 只要setUp执行成功，tearDown一定会执行，和测试方法的执行结果无关。

测试套件（test suite）

测试用例的集合，一起执行。

  • tests/widget/test_widget_suite.py
  • tests/widget/test_widget.py
  • tests/widget/__init__.py

测试执行器（test runner）

编排测试用例的执行过程，并反馈结果。

兼容旧代码

class unittest.FunctionTestCase(testFunc, setUp=None, tearDown=None, description=None)

  • app/widget_func.py
  • test/widget_func/test_widget_func.py

忽略测试用例与反转结果

  • @unittest.skip忽略某些测试用例；
  • @unittest.expectedFailure期望执行结果是失败的。

利用子测试区分迭代

  • TestCase.subTest()后不会因结果失败而终止迭代，且失败时可显示subTest的参数。

异步测试

  • unittest.IsolatedAsyncioTestCase支持协程作为测试函数。
