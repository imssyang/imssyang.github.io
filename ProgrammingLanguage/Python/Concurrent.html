Python并发编程
[ Python ]

Posted on 1/15/2022 Edited on 1/23/2022 Symbols count in article: 20k Reading time ≈ 18 mins.

python中实现并发编程的三种方式及其应用场景。

concurrency（并发性）

  • Process: 多个进程可在多个CPU核心上并行执行
  • Thread: 适合抢占式多任务（preemptively multitasked），操作系统决定什么时候切换任务
    • 支持并发执行（concurrency）和并行执行（parallelism）
  • Coroutines: 适合协同式多任务（cooperatively multitasked），由任务决定什么时候放弃控制器
    • 支持并发执行（concurrency），但不支持并行执行（parallelism）
    • 协程的切换不需要调用任何系统接口，也不会阻塞其他协程的处理过程
    • 协程的切换不需要mutex,semaphore等同步原语的参与

Process-based

  • multiprocessing模块支持多进程编程，每个进程有自己独立的解释器，因此不受GIL限制，支持并行执行（Parallelism）

Thread-based

A Zoomable Interactive Python Thread Visualization
Python全局解释器锁

  • CPython解释器：
    • threading模块创建的python线程等同于系统线程（pthread/windows threads），完全由操作系统管理
    • 解释器中存在全局锁 - Global interpreter Lock(GIL)，以确保解释器只同时运行一个线程，这简化了内存管理等问题，但限制了并行执行
      • 当线程A运行时，它持有GIL；当遇到I/O（read,write,send,recv,…)时释放GIL
      • GIL是基于mutex和condition varibale实现的

Coroutine-based

  • python实现是在单个处理器上运行，因此一次只能运行一个协程

coroutine vs subroutine

  • Coroutine 协程：在非抢占式多任务中，通过suspended/resumed的方式用来生成Subroutine。
    • Melvin Conway在1958年发明此术语，用于汇编程序的构造过程中（from wiki coroutine）；
    • 可以通过调用其他协程而退出，并且后来可回到之前退出时的执行点，从协程自身的角度看，退出只是呼叫了另一个协程。
    • 协程可能同时有multiple-instance存在，每个instance可能维护着不同的状态；
    • 两个coroutine之间是symmetric的关系
  • Subroutine 子例程
    • Subroutine可以看成是一种特殊的Coroutine;
    • 有确定的入口和出口，只返回一次，在调用期间无需维护状态；
    • 两个subroutine之间是caller-callee的关系；
  • coroutine used in P/C

coroutine vs generator

  • Coroutines 协程
    • yield语句可以让执行器立即执行其他协程
  • Generator 生成器
    • generator主要用于简化迭代器的编写
    • yield语句只能将控制权返回给生成器的调用者
    • 可以在生成器基础上实现协程
      • 引入调度（dispatcher）例程后，可以通过token显式的将控制权传递给生成器
  • coroutine used in P/C

Python实现

generator-based

  • 基于generator对象实现协程：
    • 协程A通过yield返回A/generator对象的方式放弃控制权给执行器，执行器将控制权分发至协程B
    • 协程B执行结束后同A的方式放弃控制权给执行器，执行器通过A/generator对象将控制权分发至协程A
    • 周而复始…
  • NOTE:
    • 协程的切换只能通过执行器操作generator对象实现

generator对象

What does the “yield” keyword do?

  • Iterable 可迭代的，适合for...in语法
    • list/string/… 在内存中存储所有值
    • generator 迭代时动态生成值，不需要在内存中存储所有值，但只能迭代一次
      • :) generator generates electricity but does not store energy (发电机发电但不存储电能)
      • :) black blind man breaks corn (黑瞎子掰苞米 - 边干边丢)

yield关键字

python generator “send” function purpose?

  • Python2.x使用yield或gevent模块；
  • yield关键字由两个含义：
    1. 生产某物，例如 to yield corn (生产玉米)
    2. 停下来让别的事情继续，例如 cars yielding to pedestrians (车辆给行人让路)
  • yield函数返回generator对象，配合in操作可以避免迭代到末尾时arise StopIteration
  • yield&next
  • yield&send
  • yield&in
  • producer/consumer

asyncio - Event Loop

    • 协程A通过yield from（python3.3）或await直接放弃控制权给另一个协程B

asyncio-basic

How does asyncio actually work?

  • Python3.4引入asyncio模块
    • 协程函数（coroutine function）: 通过async def定义的函数
      • iscoroutinefunction()判断是否为协程函数
    • 协程对象（coroutine object）: 协程函数的返回值是协程对象
      • iscoroutine()判断是否为协程对象
    • 只有asyncio的事件循环运行后，协程才会被执行，且协程中调用time.sleep()会阻塞EventLoop，影响其他协程的运行
    • @asyncio.coroutine（python3.8弃用，use async def）修饰一个函数为coroutine，因此可以在EventLoop中执行
  • Python3.5引入async/await语法；
    • 将@asyncio.coroutine替换为async
    • 将yield from替换为await
  • (X)time.sleep
  • yield from
  • await
  • async&await
  • python3.5+

并发场景

CPU-bound任务

  • 任务特征：
    • 程序在CPU操作上耗时多
    • 需要找到在单位时间内进行更多计算的方法
  • 并发结论：
    • asyncio因为只跑在一个CPU核心上，且需要管理并调度协程，不能为任务提速
    • threading版本因为GIL的存在，且需要管理并调度线程，会比sync版本还会慢一些，不能为任务提速
    • multiprocessing版本可以利用多核并行计算，将任务合理划分为独立的步骤后，可以为任务提速
  • cpu_sync
  • cpu_multiprocessing
  • cpu_threading

I/O-bound任务

  • 任务特征：
    • 程序耗时在与slow device交互，例如network,hard drive,printer,…
    • 需要重叠等待I/O操作的时间以加快速度
  • 并发结论：
    • 基于asyncio的协程方式效果最好，但要求依赖的模块要支持async的实现，且需注意：
      如果某个协程执行时出现了长时间阻塞，执行器里的所有协程都会受到影响
    • 基于threading的线程方式要考虑Race Conditions，编码和调试复杂度较高
    • threading版本的速度快于multiprocessing版本，但都远好于不支持并发的sync版本
  • io_sync
  • io_multiprocessing
  • io_threading
  • io_asyncio
