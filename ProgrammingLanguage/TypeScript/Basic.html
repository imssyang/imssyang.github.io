TypeScript基础
[ TypeScript ]

Posted on 3/9/2023 Symbols count in article: 13k Reading time ≈ 12 mins.

TypeScript的目标是成为JavaScript的静态类型检查器，即在运行JavaScript之前确保程序的类型是正确的。

类型

  • 常用类型，可以通过typeof判断值的类型，例如(typeof obj === "object")
    • 基本类型（primitive）
      • null (typeof null由于历史原因等于 typeof object)
      • undefined
      • string
      • number
      • bigint
      • boolean
      • symbol
    • 扩展类型
      • void 表示函数没有返回值，例如 function f() { return; }
      • object 表示非primitive值，不同于全局类型Object
      • Function 全局类型，描述类似bind, call, apply的属性
    • 特殊类型：
      • any 用在不希望出现类型检查错误的地方，编译器无法推断类型时也会默认any
      • unknown 类似any，但对此类型的值做任何操作都是非法的
      • never 用于表示union类型在选择逻辑中已经收窄（narrowing）到空的状态
        • never类型可以赋值给任何类型，但never类型不接受任何类型赋值给它
    • 泛型类型
      • Array: [1, 2, 3]
      • ReadonlyArray或readonly number[]
      • Tuple：固定长度的数组，且每个位置的类型也是确定的
        • [string, number] 二元组
        • [string, number, string] 三元组
  • 操作符
    • in 判断对象是否有属性，例如if ("swim" in animal) return animal.swim();
    • instanceof 检查值是否为类型的实例，例如if (x instanceof Date) x.toUTCString();
  • never
  • Tuple

类型注解

  • 类型注解（type Annotation）用于声明变量、函数参数、函数返回值的类型，会被编译器删除
    • 支持自定义Object类型作为注解，例如pt: { x: number; y: number }
    • 支持字面量作为注解，例如alignment: "left" | "right" | "center"
    • 支持函数类型表达式（类似箭头函数）作为注解，例如fn: (a: string) => void
    • ?: 用在函数参数上，说明参数是可选的，例如id?: number
    • keyof 可以将某个对象的key收集起来作为类型
    • in keyof 可将某个对象的所有属性收集起来
  • General
  • ByObjectType
  • OptionalProperties
  • LiteralType
  • FunctionType
  • keyof

联合类型

  • 联合类型（union type）是指多种类型的集合
    • | 操作符，例如id: number | string

类型别名

  • 类型别名（Type Alias）用于定义类型别名，便于重用
    • type 关键字，例如type ID = number | string;
    • & 操作符进行类型扩展
  • General
  • Extends

声明接口

  • 声明接口（interface declaration）是命名对象类型（object type）的一种方法
    • extends关键字进行接口扩展，返回boolean值
    • readonly 标识属性是只读的，例如interface SomeType { readonly prop: string; }
      • 只表示属性自身immutable，其内部的值仍然是可变的
    • 索引签名（index signature）支持描述对象的结构，例如{ [key: string]: number }
    • 支持重新声明以新增字段
  • General
  • Extends
  • IndexSignature
  • AddNewField

类型断言

  • 类型断言（type assertion）用于指定更具体的返回值类型，会被编译器删除，因此不影响js运行时状态
    • as关键字
      • as const后缀转换整个对象（包括每个属性）作为字面量类型（literal type）
    • <>语法
    • !后缀 表示值非null或undefined时才会执行后续操作

函数

  • 函数类型表达式（function type expression）用来描述函数，类似于箭头函数
    • 无法描述带属性（property）的函数对象
  • 调用签名（call signature）通过类型别名的方式描述带属性的函数
  • 构造签名（construct Signature）用来描述通过new操作符调用的函数
  • 重载签名（overload signature）支持描述以多种方式来调用的函数
  • FunctionTypeExpression
  • CallSignature
  • ConstructSignature
  • OverloadSignature

泛型函数

  • 泛型函数（generic Function）
    • 支持类型推理（inference）
    • 支持类型约束（constraint）
    • 支持指定类型参数（Specifying Type Arguments）
  • Basic
  • WithInference
  • Constraint
  • SpecifyingTypeArgument

模块

  • TS和ES6标准一样，文件中出现import或export语句时，此文件即为module，反之文件作用域是global的。

声明文件

声明文件（Declaration File, “.d.ts”）用于为JavaScript编写的API提供TypeScript类型信息。例如jquery.js为JS库，可以新增一个jquery.d.ts文件，里面以TS语法声明用到的API，在index.ts中调用jquery时可以享受到TS语法的支持，如同jquery本身是TS实现的一样。

  • util.js
  • util.d.ts
  • index.ts

导入库

  • 两种导入依赖方式：
    • reference注释：适用于出现global库时
    • import语句：支持module
  • reference
  • import

编译器

tsc CLI Options

  • tsc 将ts代码转换为js

    • noImplicitAny 用error代替隐式any类型
    • strictNullChecks 使用值之前必须确保它不是null或undefined
    • -p, --project 指定项目root目录（tsconfig.json文件所在的目录）

  • 为js自动生成d.ts文件（js -> d.ts）：
    npx -p typescript tsc src/**/*.js --declaration --allowJs --emitDeclarationOnly --outDir types
