编程语言概述
[ ProgrammingLanguage ]

Posted on 6/13/2021 Edited on 7/3/2021 Symbols count in article: 8.7k Reading time ≈ 8 mins.

编程语言在演化过程中，抽象层次逐步递增，演化出各种编程范式，这些专有名词常常出现，为了更好的理解这些内容，理清其中的脉络，整理出这篇概述性的文章。

演化

每一代的定义随着编程风格的日益强大逐渐演化，定义也随之进行修正。

  • 第一代（1GL, First Generation）
    • 机器层次，编码更适合机器理解，编码直接被CPU执行，速度快，但难以修复错误且可移植性低。
    • 通过前置面板的开关控制进行机器码编程。目前在驱动、固件接口以及硬件设备领域仍有应用。
  • 第二代（2GL, Second Generation）
    • 针对CPU指令编程，因此代码可被人读写，必须汇编（assembly）为机器码后才可运行。
    • 由于需要关注更多硬件细节，因此容易产生bug，且编码特定于具体的CPU架构。目前在内核、驱动、视图等领域应用。
  • 第三代（3GL, Third Generation）
    • 追求更好的可移植性（machine-independent）和易于人理解（programmer-friendly）的编码；
    • 需要编译（compilation）或解释（interpretation）执行，抽象层级更高；
    • 在1950年代首次出现，标识为算法（algorithmic）或过程（procedural）语言。例如C、C++、Java、JavaScript等。
  • 第四代（4GL, Fourth Generation）
    • 强调相对3GL进步的地方，更高层次的抽象硬件细节、更易于人理解、更强大和更通用，强调对数据集的操作，而不是比特（bit）或字节（byte）。
    • 概念在1970~1990年代与3GL同时发展，标识为非过程（non-procedural）或程序生成（program-generating）语言。
    • 主要应用于数据库管理、报告生成、视图开发和web开发领域，例如SQL。
    • 一些增强的3GL语言，例如Python、Ruby和Perl在3GL环境下扩展了一些4GL能力。
  • 第五代（5GL, Fifth Generation）
    • 基于程序的约束条件解决问题，而非算法。用户只需要考虑需要解决的问题以及需要满足的条件，无需实现程序（routine）或算法。
    • 在1980年代出现，例如声明式语言（declarative languages）。
    • 目前应用于人工智能（AI，artificial intelligence）搜索领域。

范式

编程范式（Programming paradigms）是根据语言特性进行分类的方法。一种语言可以支持多种编程范式，分类依据如下：

  • 句法（syntax）和语法（Grammar）风格。
  • 执行模型（execution model）：工作顺序、是否允许副作用（side effects）等。
    • 执行模型决定了工作单元被调度（scheduled）执行的方式，例如工作单元执行的顺序，
    • 通常通过编译器（compiler）或者解释器（interpreter）实现，包含一个运行时系统（runtime system）。
      • 静态选择（static choice）指工作顺序在执行前已经确定，由编译器实现，工作顺序取决于指令放入可执行文件中的顺序；
      • 动态选择（dynamic choice）指工作顺序在执行期间确定，在运行时系统或者解释器中实现。
    • 另外，执行模型也可以独立于编程语言存在，例如POSIX Threads库。
  • 代码组织方式：面向对象。

命令式编程

命令式编程（Imperative programming）：是面向硬件的抽象，程序员指示机器如何改变它的状态（state）。

硬件抽象 图灵机 示例
变量 存储状态的单元 x=1;x=2;
表达式 内存引用和算术运算 5+3、&x
赋值语句 get/put指令 x=x+1
控制语句 jump指令 if、for

过程式和面向对象编程中的专有名词有相似的语义，但命名方式不同：

Procedural Object-oriented
Procedure Method
Record Object
Module Class
Procedure call Message

过程式编程

过程式编程（Procedural programming）：强调将指令（instruction）分组为过程（procedures）。

  • 过程（a type of routine or subroutine）可以在程序执行期间的任何时候被其他过程或程序本身调用。
  • 1957-1964年首次出现，包括Fortran、ALGOL、COBOL、PL/I和BASIC。1970-1972年Pascal和C语言发布。

面向对象编程

面向对象编程（OOP, Object-oriented programming）：强调将指令（instruction）与其操作的状态（state）一起打包为对象（object）。

声明式编程

声明式编程（Declarative programming）：程序员将期望的结果声明出来，但不关注如何计算它。

函数式编程

函数式编程（Functional programming）：是面向数学的抽象，期望的结果是一系列函数应用通过lambda演算（λ-calculus）得到的值。1950年代在Lisp语言中首次开发使用，其不擅长处理可变状态和IO。

数学概念 含义
变量 一个值的名称，且x=x+1无意义 不可变的（immutable）、不允许赋值语句。
函数（Subroutine） 自变量的映射（y=f(x)，其中x为自变量，y为因变量） 一个函数的值仅由函数参数决定，不依赖其他状态。
函数作为一等公民（first-class citizen） 函数可以在任何地方定义、作为函数的参数和返回值、对函数进行组合等一切操作。
递归（Recursion） y=f(x),其中0<x<10,求y=1000时的x值。每次计算使y=x直至x合法。 函数不断调用自己，直到达到基本条件，可看成对循环语句的模拟。
lambda演算（λ-calculus） (x=1,y=3时) z=2x+y+1=2*1+3+1=6 -> z=(λx.(λy.2x+y+1)3)1=(λy.2*1+y+1)3=2*1+3+1=6 使用λ标记函数参数，完整抽象了计算机中的函数概念，是图灵完备（Turing completeness）的，从而使得函数式编写的程序可被编译成冯诺伊曼体系结构中机器指令执行。

特性：

  • 程序是通过应用（applying）和组合（composing）函数进行构造的，函数式编程语言一般提供强大的模式匹配（Pattern Match）；

  • 纯函数（Pure functions, also expression）是数学函数的计算模拟。

    • 对于相同的参数，函数返回值是相同的（可重入的）；
    • 该函数应用程序没有任何副作用（Memory/IO）；
    • 纯函数的运算是线程安全（thread-safe）的，意味着如果两个纯函数没有数据依赖，可以并行（parallel）执行。
      # Pure
void f() {
  static std::atomic<int> x = 0;
  ++x;
}

# Impure: because of mutation of a local static variable
void f() {
  static int x = 0;
  ++x;
}

# Impure: because of return value variation with a mutable reference argument
int f(int* x) {
  return *x;
}

# Impure: because of mutation of an output stream
void f() {
  std::cout << "Hello, world!" << std::endl;
}

  • 递归（Recursion）函数调用自己，直到达到基本条件。函数式编程中通过递归来完成迭代(循环)。

    • 尾递归（tail-recursion）是指递归操作是子例程的最后一步，此时当前过程中的数据不再被需要，无需放入堆栈，因此有利于编译器优化为无需调用栈的实现。
      unsigned int factorial(unsigned int n) {
    return (n == 0) ? 1 : n * factorial(n - 1);
}

  • 高阶函数（Higher-order functions）是指可以将函数作为参数或者返回值的函数，使得复用粒度降低到函数级别。

    • c++
    • java
    • c#
    • go
    • javascript

  • 柯里化（currying）是一种将多个参数的函数转换为单个参数的函数序列的技术，是为了适配lambda演算过程要求只有一个参数而发明。

    x = f(a,b,c) becomes:
  h = g(a)
  i = h(b)
  x = i(c)
called in sequence:
  x = g(a)(b)(c)

  • 惰性求值（non-strict/lazy evaluation）只有当函数内的计算需要使用参数的值时，才计算函数参数。函数式编程中多使用惰性求值。严格求值（strict evaluation）总是在调用函数之前完全计算函数参数。

    # 严格求值时报除0错误，惰性求值时得到数组长度为4.
print length([2+1, 3*2, 1/0, 5-4])

  • 引用透明（Referential transparency）是指变量可以在执行期间的任何时间点用真实值取代，而不会影响结果。函数式编程中变量一旦被定义就不会再改变，因此不支持赋值语句，函数式编程都是引用透明的。

    # x初始为1时，连续两次执行赋值语句分别得到2和3。赋值语句从来不是引用透明的。
x = x + 1

# x初始为1时，连续两次执行函数调用分别得到2和2。函数是透明的，不会隐式改变x，没有副作用。
int plusone(int x) { return x+1; }

  • 闭包（Closure）是一种在支持函数作为一等公民的编程语言中实现词法作用域内进行名称绑定的技术，它同时存储了函数与其环境。这个概念是在lambda演算（λ-calculus）过程中用于评估表达式而设计。

    # python中匿名函数实现闭包
def f(x):
    def g(y):
        return x + y
    return g  # Return a closure.

def h(x):
    return lambda y: x + y  # Return a closure.

# Assigning specific closures to variables.
a = f(1)
b = h(1)

# Using the closures stored in variables.
assert a(5) == 6
assert b(5) == 6

# Using closures without binding them to variables first.
assert f(1)(5) == 6  # f(1) is the closure.
assert h(1)(5) == 6  # h(1) is the closure.

逻辑式编程

逻辑式编程（Logic programming）：是基于形式逻辑（formal logic）的抽象，期望的结果是由事实（fact）和规则（rule）构成的一些条款（clause）。

  • 1960-1970年代讨论人工智能（artificial intelligence）时出现，包括Prolog、Datalog。

响应式编程

响应式编程（Reactive programming）期望的结果是用数据流（data stream）和传播变化（propagation of change）来声明的。

// 命令式编程不会传播变化
var b = 1
var c = 2
var a = b + c
b = 10
console.log(a) // 3 (not 12 because "=" is not a reactive assignment operator)

// 想象"$="除了支持在初始时赋值，也支持在右边变化时重新赋值
var b = 1
var c = 2
var a $= b + c
b = 10
console.log(a) // 12 (because "$=" is a imagine reactive assignment operator)
  • 应用在硬件描述语言，比如Verilog，可以通过电路构建传播模型。
  • 在model–view–controller (MVC)架构中，底层模型（model）中的更改会自动反应到关联的视图（view）中。

反射编程

反射编程（reflective programming）是支持检查、修改自身数据结构与行为的能力。

  • 早期的汇编语言天生支持此能力，后续的高级语言逐渐消失，直至内置反射类型系统的语言出现。
  • 反射特别适合以同样方式处理不同数据的场景，比如数据的序列化、绑定、展示、转换。对于网络应用、运行时检查或修改程序行为、软件测试（运行时实例化对象）十分便利。

并发编程

并发编程（Concurrent programming）适用于有并发模型的语言，涉及多线程（multi-threading）、分布式计算（distributed computing）、消息传递（message passing）、共享资源（shared resources）、期望与承诺（future&promise）。

角色模型

角色模型（Actor model）在计算机科学中是一种用于并发计算的数学模型。

  • 1973年收到物理学启发，应对高并行计算的需要而设计。
  • 当角色收到消息时，可能的行为有：本地决策、创建其他角色、发送消息、考虑如何处理下一条消息；
  • 每个角色可以有自己的私有状态，但状态改变时只能通过消息在角色间相互传递。因为没有共享的状态，所以不需要依赖锁进行同步（lock-based synchronization）；
# <基于Actor模型构建简单的存储单元>
# Actor对象: 存储单元
Cell ≡
    receiver
        Request[Create[initial] customer]
            send customer Returned[serializer ReadWrite(initial)]
# Actor行为：读写数据
# - behavior是流水线式的，当后续的read/write消息到达时，behavior可能正处理之前的消息；
ReadWrite(contents) ≡
    behavior
        Request[read[] customer]
            {send customer Returned[contents], ReadWrite(contents)}
        Request[write[x] customer]
            {send customer Returned[], ReadWrite(x)}

# 示例：创建一个初始值为5的Actor对象x，然后并发的向x写入7和9，最后读取x的值。
# （x最终的结果可能为5、7或9）
let x = Cell.Create[5] in {x.write[7], x.write[9], x.read[]}

泛型编程

泛型编程（Generic programming）是指编写时以通用类型表示算法，后续根据传参按需进行实例化类型的风格。

  • 1973年在ML语言中首创，运行编写通用类型的函数以减少重复，也被称为参数多态性（Parametric polymorphism）；
  • 1989年由Musser & Stepanov (1989)定义了泛型编程（Generic programming）概念：从具体的算法中抽象出泛型算法，再与各种数据结合产生有用的软件；
  • 因C++中的标准模板库（STL, Standard Template Library）而闻名，使用迭代器（Iterator）解耦了序列化数据和操作它们的算法；
  • 数组（Array）和结构体（Struct）也可以看作预定义的泛型类型；
    • 数组或结构类型的每次使用都会实例化一个新的具体类型，或重用先前实例化的类型。
    • 数组元素类型和结构元素类型是参数化类型（parameterized type），用于实例化相应的泛型类型（generic type）。
    • 所有这些通常都是编译器内建的，其语法与其他泛型结构不同。

模板元编程

模板元编程（TMP, Template metaprogramming）是指编译器使用模板生成临时源代码，并将其与源代码的其余部分合并，然后进行编译的技术。

动态编程

动态编程语言（Dynamic programming language）是指在运行时执行静态编程语言在编译期间执行的许多常见编程行为。

  • 大部分动态语言也是动态类型的（dynamically typed），也经常被称为脚本语言（scripting languages），即使脚本语言狭义上的定义为特定于运行环境的语言。
  • Eval函数：一些动态语言会提供eval函数，接受一个包含该语言中的代码的字符串参数并执行它，如果此代码代表一个表达式，则返回结果值。
  • 对象运行时改变（Object runtime alteration）：在动态语言中，类型或对象系统通常可以在运行时进行修改。
  • 反射（Reflection）：反射在许多动态语言中都很常见，通常涉及对泛型或多态数据的类型和元数据的分析。
  • 宏命令（Macros）：有些动态语言将代码内省和eval进行结合，提供提供对编译器内部工作方式的访问，以及对解释器、虚拟机或运行时的完全访问，允许定义可以优化代码或修改语言语法的结构（Assembly、C、c++、早期Java一般不属于这一类）。
  • 流行的动态编程语言包括JavaScript、Python、Ruby、PHP、Lua和Perl。

事件驱动编程

事件驱动编程（Event-driven programming）是指程序的流程由用户操作(鼠标点击、按键)、传感器输出或其他程序或线程传递的消息等事件决定的。

  • 通常有一个主循环侦听事件，然后在检测到其中一个事件时触发回调函数。
  • 在嵌入式系统中，可以使用硬件中断而不是持续运行的主循环来实现相同的目的。
  • 事件驱动的程序在await和闭包（closures）等高层次抽象的语言中实现更简单。
