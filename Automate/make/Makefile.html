变量（variable）

Make

  • 变量只支持字符串，未定义变量为空字符串
    • = 使用此变量时，其左侧的值才会被解析
    • := 定义此变量时，其左侧的值才会被解析
    • ?= 变量未定义时，才会赋值给它
    • += 追加字符串
    • override 允许覆盖来自命令行参数中的变量
    • \ 反斜杠字符支持将长字符串写成多行
  • 单引号或双引号没有特殊意义，只标识字符串
  • 引用变量可通过${}和$()
  • basic
  • :=
  • ?=
  • +=
  • override

自动变量（automatic）

10.5.3 Automatic Variables

  • $@ 目标（target）的文件名称，例如 %.c: -> xxx.c
  • $* 目标的匹配名称，例如 %.c: -> xxx
  • $< 第一个先决条件（prerequisite）的名称，例如 x.o: a.c b.c -> a.c
  • $^ 由空格分隔的所有先决条件的名称，例如 x.o: a.c b.c -> a.c b.c
  • $? 由空格分隔的所有新于目标的先决条件的名称
  • $| 由空格分隔的所有order-only的先决条件的名称
  • $% 归档（archive）目标的成员名称，例如 foo.a(bar.o): -> bar.o
  • $+ 同$^，已排序

Shell

  • 默认SHELL=/bin/sh
  • 环境变量既可作为make变量，又可作为shell变量
  • $$ 引用shell变量
  • export 将一个make变量转换为环境变量
  • EXPORT_ALL_VARIABLES 将所有make变量转换为环境变量
  • set-shell
  • make&shell-variable
  • env-variable

隐式规则（Implicit）

CC: C编译器，默认cc
CXX: C++编译器，默认g++
CFLAGS: C编译选项
CXXFLAGS: C++编译选项
CPPFLAGS: C预处理器选项
LDFLAGS: 链接选项

$(CC) -c $(CPPFLAGS) $(CFLAGS) (n.c -> n.o)
$(CXX) -c $(CPPFLAGS) $(CXXFLAGS) (n.cc/n.cpp -> n.o)
$(CC) $(LDFLAGS) n.o $(LOADLIBES) $(LDLIBS) (n.o -> n)
