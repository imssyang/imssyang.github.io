Github持续集成与交付
[ github ]

Posted on 6/16/2022 Symbols count in article: 3.7k Reading time ≈ 3 mins.

Github 仓库已支持 CI/CD (continuous integration and continuous delivery) 流程，每个仓库均可实现自动化build，test，deploymemt。

Learn GitHub Actions

流程

  1. 部署runner
  2. 在repository中编写需要自动化的workflow
  3. 在workflow中指定event触发方式
  4. 在workflow中指定运行所在的runner
  5. 在workflow中指定每个步骤的job及其依赖关系
  6. 在workflow中编写每个job使用的action及shell命令
  7. 提交workflow后，自动交由Github Actions平台处理
  8. event产生时，runner拉取repository代码，然后执行job

workflow

Workflow syntax
Assigning permissions to jobs
Using concurrency

  • 脚本目录：.github/workflows/*.yml，文件为YAML语法
  • 一个repository可以有多个workflow，且workflow可以互相调用以复用代码
  • 一个yml文件对应一个workflow，一个workflow可包含多个job

event

Events that trigger workflows

  • event用于触发workflow执行，支持git操作、REST/API、manual方式

job

  • job是在workflow中定义在runner上执行的最小单位，支持shell语法
  • job是顺序执行的且多个命令相互独立执行
  • job间可以有依赖关系，没有依赖关系的job会并行执行

action

Metadata syntax for GitHub Actions
GitHub Marketplace Actions

  • action是在Github Actions平台上执行的特定应用，打包了一些频繁使用且复杂的重复task
  • job中使用action可减少重复的代码量
  • 通过uses引用action，支持形式有
    • 公共仓库中的名字：actions/checkout@v3
    • 仓库中的相对目录：./.github/actions/demo
    • docker镜像地址：docker://alpine:3.8
  • 支持自定义action

runner

  • runner是用于执行workflow的服务

关键字

  • '-' - 分割step
  • env - 自定义环境变量
  • name - job/step的名字
  • needs - 依赖的job
  • run - 执行shell
  • users - 执行action

内置对象

Contexts

  • github - workflow运行相关信息
  • env - workflow中的环境变量
  • job - 当前job信息
  • steps - 当前job中的steps信息
  • runner - 当前job所在的runner信息
  • secrets - workflow可用的私密信息
  • strategy - 当前job的执行策略
  • matrix - 当前job的矩阵属性
  • needs - 当前job依赖的其他job的输出信息
  • inputs - workflow的输入信息

环境变量

Environment variables

  • 三种类型：
    • env：应用整个workflow
    • jobs.<job_id>.env：应用特定job
    • jobs.<job_id>.steps[*].env：应用特定step
  • 支持通过context读写环境变量
    • ${{ env.MY_VARIABLE }} 通过env访问自定义环境变量
  • 自定义路径相关的环境变量应该加_PATH后缀

表达式

Expressions

  • ${{ }}格式表示作为表达式解释，而非字符串
  • 当if条件（默认按表达式解释）为ture时，step将运行
  • 支持数据类型：boolean, null, number, string（单引号作为转义字符）
  • 支持内建函数
    • contains: 包含子项时为true （大小写不敏感）
    • startsWith: 以特定字符开始时为true（大小写不敏感）
    • endsWith：以特定字符结束时为true （大小写不敏感）
    • format：按占位符格式化字符串
    • join：连接两项
    • toJSON：序列化json字符串
    • fromJSON：反序列化json字符串
  • 支持状态检查函数：
    • success：之前的step既没有失败又没有取消时为true
    • failure：之前的step失败时为true
    • cancelled：workflow被取消时为true
    • always：step始终执行时为true
